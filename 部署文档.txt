一、项目概述
================================================================================

本项目是一个基于华为 HarmonyOS（鸿蒙）的校园社团管理系统，采用 ArkTS
语言开发，使用 Stage 模型架构。系统包含三种角色：学生、社团负责人、管理员，
涵盖社团管理、活动管理、财务管理、成员管理、数据统计等功能。

技术栈：
  - 开发语言：ArkTS（基于 TypeScript 的鸿蒙开发语言）
  - 开发框架：ArkUI 声明式 UI 框架
  - 架构模式：Stage 模型
  - 目标平台：HarmonyOS NEXT
  - SDK 版本：HarmonyOS SDK 6.0.0(20)（兼容最低 5.0.5(17)）
  - 构建工具：Hvigor 6.0.1
  - 开发工具：DevEco Studio 5.0 或更高版本
  - 数据服务：本地 MockDataService 模拟端云协同（可独立运行，无需云端）

项目目录结构：
  MyApplication3/                    ← 【主项目，请打开此文件夹】
  ├── AppScope/                      应用全局配置
  │   └── app.json5                  应用基本信息（包名、版本等）
  ├── entry/                         主模块
  │   └── src/main/
  │       ├── ets/                   ArkTS 源代码
  │       │   ├── common/            常量定义
  │       │   ├── entryability/      入口 Ability
  │       │   ├── models/            数据模型（7个）
  │       │   ├── pages/             页面文件（18个）
  │       │   ├── services/          数据服务（MockDataService）
  │       │   ├── store/             状态管理
  │       │   ├── theme/             主题配置
  │       │   └── utils/             工具类
  │       ├── resources/             资源文件
  │       └── module.json5           模块配置
  ├── build-profile.json5            构建配置
  ├── oh-package.json5               包管理配置
  └── 测试账号                       测试账号信息

  CloudProgram/                      ← 云端参考代码（无需部署，仅作展示）
  ├── clouddb/schema.json            云数据库表结构定义
  └── cloudfunctions/                AGC 云函数参考实现
      ├── user-auth/                 用户认证云函数
      ├── club-manage/               社团管理云函数
      ├── activity-manage/           活动管理云函数
      └── finance-manage/            财务管理云函数

二、环境要求
================================================================================

【操作系统】
  - Windows 10（64位）或更高版本
  - macOS 12 或更高版本（Intel / Apple Silicon 均可）

【硬件要求】
  - 内存：建议 16GB 及以上（最低 8GB）
  - 磁盘空间：至少 20GB 可用空间（IDE + SDK + 模拟器）
  - CPU：x86_64 架构，建议 Intel i5 或同等性能以上

【软件要求】
  - DevEco Studio 5.0.0 Release 或更高版本（推荐最新版）
  - HarmonyOS SDK（通过 DevEco Studio 自动下载）
  - Node.js 不需要单独安装（DevEco Studio 内置）
  - Java：不需要单独安装（DevEco Studio 内置 OpenJDK）

【网络要求】
  - 首次安装 DevEco Studio 和 SDK 时需要联网下载
  - 项目运行时需要联网（部分页面加载在线图片资源）

三、开发工具下载与安装（DevEco Studio）
================================================================================

DevEco Studio 是华为官方推出的 HarmonyOS 应用开发 IDE，基于 IntelliJ 
平台，类似 Android Studio。

【步骤 1】下载 DevEco Studio

  打开浏览器，访问华为开发者官网下载页面：
  https://developer.huawei.com/consumer/cn/deveco-studio/

  页面上会显示最新版本的 DevEco Studio，点击对应操作系统的下载按钮：
    - Windows 版：下载 .exe 安装包（约 1.5GB）
    - macOS 版：下载 .dmg 安装包

  注意：如果页面要求登录华为账号，请先注册一个华为开发者账号（免费）。
  注册地址：https://id.cloud.huawei.com/CAS/portal/registerCode.html

【步骤 2】安装 DevEco Studio（Windows）

  2.1  双击下载的 .exe 安装包，启动安装向导
  2.2  点击"Next"
  2.3  选择安装路径（建议使用默认路径，或选择一个英文路径，不要含中文！）
       例如：C:\Program Files\Huawei\DevEco Studio
  2.4  勾选以下选项：
       ☑ Create Desktop Shortcut（创建桌面快捷方式）
       ☑ Add "Open Folder as Project" to context menu（可选）
       ☑ Add to PATH（添加到系统环境变量）
  2.5  点击"Install"开始安装，等待安装完成
  2.6  安装完成后，勾选"Run DevEco Studio"，点击"Finish"

【步骤 3】安装 DevEco Studio（macOS）

  3.1  双击下载的 .dmg 文件
  3.2  将 DevEco Studio 图标拖拽到 Applications 文件夹
  3.3  从 Launchpad 或 Applications 文件夹启动 DevEco Studio

【步骤 4】首次启动配置

  4.1  首次启动 DevEco Studio，会弹出欢迎界面
  4.2  如果提示导入设置（Import Settings），选择"Do not import settings"
  4.3  接受隐私协议和许可协议，点击"Continue"
  4.4  进入"Customize DevEco Studio"页面：
       - UI Theme：选择喜欢的主题（Light 或 Dark）
       - 点击"Next"

  4.5  进入 SDK 安装配置页面（非常重要！请看下一章节）


================================================================================
四、HarmonyOS SDK 配置
================================================================================

首次启动 DevEco Studio 时，会自动引导安装 HarmonyOS SDK。

【步骤 1】SDK 路径设置

  在 SDK Setup 页面：
  - HarmonyOS SDK Location：选择 SDK 安装路径
    建议使用默认路径：
      Windows: C:\Users\<你的用户名>\AppData\Local\OpenHarmony\Sdk
      macOS:   /Users/<你的用户名>/Library/OpenHarmony/Sdk
    
    ⚠ 注意：路径中不能包含中文、空格或特殊字符！

  - 点击"Next"

【步骤 2】SDK 组件选择

  确保以下组件被勾选：
    ☑ HarmonyOS NEXT（API 12 及以上）
    ☑ Toolchains（工具链）
    ☑ Previewer（预览器）
    ☑ 模拟器相关组件（如有）

  点击"Next"，然后点击"Accept"接受 SDK 许可协议

【步骤 3】等待 SDK 下载安装

  SDK 下载大小约 3-5GB，根据网络速度可能需要 15-60 分钟。
  请保持网络畅通，耐心等待下载完成。

  下载完成后，点击"Finish"进入 DevEco Studio 主界面。

【步骤 4】验证 SDK 安装

  4.1  在 DevEco Studio 主界面，点击菜单栏：
       File → Settings（Windows）
       DevEco Studio → Preferences（macOS）

  4.2  在左侧导航栏找到：
       HarmonyOS SDK

  4.3  确认 SDK 路径正确，且显示已安装的 API 版本
       需要确保有 API 12 或更高版本

  4.4  如果 SDK 未安装或版本不够，点击"Edit"按钮进行安装或更新

六、使用 DevEco Studio 打开项目
================================================================================

【步骤 1】启动 DevEco Studio

  双击桌面上的 DevEco Studio 快捷方式，或从开始菜单启动。

【步骤 2】打开项目

  方式一：在欢迎页面点击"Open Project"
  方式二：菜单栏 File → Open

  在弹出的文件选择窗口中，导航到项目所在路径，
  选择 MyApplication3 文件夹（注意是选择 MyApplication3 这一层！）：

    D:\HarmonyProjects\MyApplication3

  点击"OK"或"Open"。

  ⚠ 不要打开 MyApplication3 内部的子文件夹（如 entry），
     必须选择 MyApplication3 根目录！

【步骤 3】等待项目加载

  打开项目后，DevEco Studio 会自动执行以下操作：
  (1) 解析项目配置文件（build-profile.json5、oh-package.json5 等）
  (2) 同步项目依赖（自动执行 ohpm install）
  (3) 构建索引

  右下角会显示进度条，请等待所有后台任务完成。
  首次打开可能需要 2-5 分钟。

【步骤 4】确认项目加载成功

  加载成功后：
  - 左侧"Project"面板应显示完整的项目目录树
  - 底部"Build"面板无红色错误信息
  - 右下角无持续的加载提示

  如果出现错误提示"SDK version not found"之类，
  请参考第四章节重新安装对应版本的 SDK。

七、项目依赖安装
================================================================================

通常打开项目时 DevEco Studio 会自动安装依赖。如果没有自动安装，
请手动执行：

【方式一】通过 IDE 操作

  菜单栏 → File → Sync and Refresh Project
  或者点击工具栏上的"Sync Project"按钮（带刷新图标的按钮）

【方式二】通过终端操作

  在 DevEco Studio 底部打开"Terminal"面板，执行：

  ohpm install

  等待安装完成，应显示：
  install completed in xx ms
  info: packages installed successfully

  注意：ohpm 是鸿蒙的包管理工具（类似 npm），已随 DevEco Studio 安装。
  如果提示"ohpm not found"，请在 Settings → Build, Execution, Deployment 
  → HarmonyOS SDK → Tools 中确认 ohpm 已安装，并且路径已添加到系统环境变量。

八、签名配置
================================================================================

HarmonyOS 应用在运行到模拟器或真机前，需要配置签名。

════════════════════════════════════════
方式一：自动签名（推荐，最简单）
════════════════════════════════════════

【步骤 1】登录华为开发者账号

  菜单栏 → File → Settings（Windows）/ Preferences（macOS）
  左侧导航 → HarmonyOS SDK → 找到 Sign In 区域
  
  或者直接在菜单栏 → File → Project Structure → Signing Configs
  
  点击"Sign In"，使用华为开发者账号登录。
  如果没有账号，请先在 https://developer.huawei.com 注册。

  注意：普通华为账号即可，不需要企业认证，个人开发者免费注册即可。

【步骤 2】配置自动签名

  2.1  菜单栏 → File → Project Structure
  2.2  左侧选择 "Signing Configs"
  2.3  勾选 "Automatically generate signature"（自动生成签名）
  2.4  选择已登录的华为账号
  2.5  选择 Product 为 "default"
  2.6  点击 "Apply" 然后 "OK"

  DevEco Studio 会自动生成调试签名证书，可能需要等待几秒钟。

  ✅ 完成后，build-profile.json5 中的 signingConfigs 会自动填充。

════════════════════════════════════════
方式二：模拟器免签名（部分情况）
════════════════════════════════════════

  在某些 DevEco Studio 版本中，使用本地模拟器运行时可能不需要签名。
  但建议还是配置自动签名，避免后续问题。

九、模拟器配置与运行
================================================================================

═══════════════════════
方式 A：使用本地模拟器
═══════════════════════

【步骤 1】打开设备管理器

  菜单栏 → Tools → Device Manager
  或点击工具栏上的设备管理器图标（手机图标）

【步骤 2】创建模拟器

  2.1  点击左下角的"+"号或"Create Device"按钮
  2.2  选择设备类型：Phone（手机）
  2.3  选择一个设备型号，例如：
       - Huawei Phone (推荐)
       - 或列表中任意手机设备
  2.4  点击"Next"
  2.5  选择系统镜像：
       - 选择 HarmonyOS NEXT 对应的系统镜像
       - 如果没有已下载的镜像，点击"Download"下载
       - 下载大小约 2-4GB，需要等待
  2.6  配置模拟器参数（可保持默认）：
       - Device Name: 可自定义名称
       - RAM: 建议 4096MB 或更高
  2.7  点击"Finish"创建模拟器

【步骤 3】启动模拟器

  在 Device Manager 中，找到刚创建的模拟器，点击右侧的"▶"（启动按钮）。
  模拟器首次启动较慢，可能需要 1-3 分钟，请耐心等待。
  启动成功后会弹出一个模拟器窗口，显示 HarmonyOS 桌面。

【步骤 4】运行项目到模拟器

  4.1  在 DevEco Studio 工具栏上方，确认：
       - 左侧下拉框选择了模块 "entry"
       - 右侧设备下拉框选择了刚启动的模拟器设备名称

  4.2  点击工具栏上的绿色三角形"▶"按钮（Run）
       或使用快捷键：Shift + F10（Windows）/ Control + R（macOS）

  4.3  等待编译和安装：
       - 底部 "Run" 面板会显示编译进度
       - 编译成功后，应用会自动安装到模拟器并启动
       - 首次编译可能需要 1-3 分钟

  4.4  运行成功标志：
       - 模拟器上显示应用启动页面（校园社团管理系统 Logo + 加载动画）
       - 约 1.5 秒后自动跳转到登录页面
       - 底部 "Run" 面板显示 "Launch successful"

═══════════════════════

十、真机调试运行
================================================================================

如果有华为/荣耀手机（系统为 HarmonyOS NEXT 或 HarmonyOS 4.0+），
可以直接在真机上运行调试。

【步骤 1】手机开启开发者模式

  1.1  打开手机"设置"
  1.2  进入"关于手机"
  1.3  连续点击"HarmonyOS 版本号"7 次
       提示"您已进入开发者模式"
  1.4  返回"设置"主页，进入"系统和更新" → "开发者选项"
  1.5  开启以下开关：
       ☑ 开发者选项（总开关）
       ☑ USB 调试
       ☑ 仅充电模式下允许 ADB 调试（如有此选项）

【步骤 2】连接手机到电脑

  2.1  使用 USB 数据线连接手机和电脑
  2.2  手机上会弹出"允许 USB 调试吗？"对话框
  2.3  勾选"始终允许来自此计算机的调试"
  2.4  点击"允许"

【步骤 3】确认设备连接

  3.1  在 DevEco Studio 工具栏的设备下拉框中，
       应该能看到已连接的真机设备名称
  3.2  如果看不到设备，尝试：
       - 重新插拔 USB 线
       - 更换 USB 接口
       - 安装华为手机驱动（通常 DevEco Studio 会自动安装）
       - 检查手机上是否弹出了授权确认对话框

【步骤 4】运行到真机

  4.1  在设备下拉框中选择真机设备
  4.2  点击"▶"运行按钮
  4.3  等待编译、签名、安装、启动
  4.4  手机上会自动打开应用

  注意：首次运行到真机时，可能需要在手机上确认"是否允许安装"。


================================================================================
十一、云服务端说明（CloudProgram）
================================================================================

【重要说明】

  本项目的客户端（MyApplication3）采用 MockDataService（本地模拟数据服务）
  实现所有业务逻辑，所有数据存储在内存中。
  
  这意味着：
  ✅ 不需要搭建任何后端服务器
  ✅ 不需要配置数据库
  ✅ 不需要注册 AGC（AppGallery Connect）账号
  ✅ 打开项目即可直接编译运行，所有功能均可正常使用

  项目中的 CloudProgram 文件夹包含的是云端参考代码，
  展示了如果要对接华为 AGC 云服务时，云函数和云数据库应如何设计。
  这些代码仅供参考和论文展示，不影响客户端运行。

  CloudProgram 文件夹包含：
  (1) clouddb/schema.json  — AGC 云数据库的表结构定义（9张表）
      包括：UserInfo、ClubInfo、ActivityInfo、ClubMember、
            FinanceRecord、ClubApplication、JoinApplication、
            ActivityRegistration、NoticeInfo

  (2) cloudfunctions/ — 4个 AGC 云函数参考实现
      - user-auth/index.js      用户登录、注册
      - club-manage/index.js     社团增删改查
      - activity-manage/index.js 活动增删改查
      - finance-manage/index.js  财务记录管理

  【如果需要部署云服务（高级，可选）】

  如果确实需要将 MockDataService 替换为真实的 AGC 云服务：
  (1) 注册华为开发者账号并完成实名认证
  (2) 登录 AGC 控制台：https://developer.huawei.com/consumer/cn/service/josp/agc/index.html
  (3) 创建项目和应用
  (4) 开通云数据库服务，导入 schema.json
  (5) 开通云函数服务，部署 cloudfunctions 中的 4 个函数
  (6) 下载 agconnect-services.json 配置文件到项目中
  (7) 修改客户端代码，将 MockDataService 调用替换为 AGC SDK 调用

  上述步骤较为复杂，且当前项目已可完整运行演示，
  如非必要不建议操作。

十四、常见问题与解决方案（FAQ）
================================================================================

───────────────────────────────────────────────────
Q1: 打开项目后报错"SDK version not found"
───────────────────────────────────────────────────
A: 本项目需要 HarmonyOS SDK API 12 或更高版本。
   解决方法：
   1. File → Settings → HarmonyOS SDK
   2. 检查已安装的 SDK 版本
   3. 如果没有 API 12+，点击 "Edit" 安装
   4. 安装完成后重新打开项目

───────────────────────────────────────────────────
Q2: 项目编译报错"Cannot find module..."
───────────────────────────────────────────────────
A: 依赖未正确安装。
   解决方法：
   1. 打开 DevEco Studio 底部的 Terminal
   2. 确认当前目录是项目根目录（MyApplication3）
   3. 执行命令：ohpm install
   4. 等待安装完成后，重新编译

───────────────────────────────────────────────────
Q3: 点击 Run 后报错"No device found"
───────────────────────────────────────────────────
A: 没有可用的运行设备。
   解决方法：
   1. 如果用模拟器：Tools → Device Manager → 创建并启动模拟器
   2. 如果用真机：确认 USB 连接正常且已开启 USB 调试
   3. 确认工具栏设备下拉框中已选择目标设备

───────────────────────────────────────────────────
Q4: 签名报错"Sign config is not configured"
───────────────────────────────────────────────────
A: 未配置应用签名。
   解决方法：
   1. File → Project Structure → Signing Configs
   2. 勾选 "Automatically generate signature"
   3. 登录华为开发者账号
   4. 点击 Apply → OK
   详见本文第八章节。

───────────────────────────────────────────────────
Q5: 应用安装到设备后闪退
───────────────────────────────────────────────────
A: 可能原因及解决方法：
   1. 设备系统版本过低：确保手机系统为 HarmonyOS 4.0 或更高
   2. 签名问题：重新配置自动签名
   3. 清理缓存：Build → Clean Project，然后重新编译运行

───────────────────────────────────────────────────
Q6: 登录后页面显示空白或数据为空
───────────────────────────────────────────────────
A: 可能是应用重启导致内存数据丢失。
   解决方法：
   1. 使用测试账号（admin/leader1/student1）登录
   2. 密码均为 123456
   3. 如果数据异常，在 DevEco Studio 中重新 Run 即可重置数据

───────────────────────────────────────────────────
Q7: 页面中的图片显示不出来
───────────────────────────────────────────────────
A: 部分图片使用了在线 URL（如 unsplash、pravatar），
   需要设备能够访问互联网。
   解决方法：
   1. 确保模拟器或真机已联网
   2. 如果是公司/学校网络，检查是否有网络代理或防火墙
   3. 图片加载失败不影响核心业务功能的使用

───────────────────────────────────────────────────
Q8: 编译速度很慢
───────────────────────────────────────────────────
A: 首次编译较慢是正常的，后续增量编译会快很多。
   加速建议：
   1. 确保电脑内存 ≥ 8GB
   2. 关闭不必要的后台程序
   3. 使用 SSD 硬盘
   4. 第二次运行会快很多（增量编译）

───────────────────────────────────────────────────
Q9: 如何查看应用运行日志？
───────────────────────────────────────────────────
A: DevEco Studio 底部面板 → "Log" 标签页
   可以筛选日志级别（Info、Warn、Error）和标签（testTag）
   应用中的 console.log/console.error 和 hilog 输出都会显示在这里。

───────────────────────────────────────────────────
Q10: 提示"ohpm: command not found"
───────────────────────────────────────────────────
A: ohpm 工具未添加到系统环境变量。
   解决方法：
   1. 找到 ohpm 安装路径（通常在 DevEco Studio 安装目录或 SDK 目录下）
      Windows 典型路径：
      C:\Users\<用户名>\AppData\Local\OpenHarmony\Sdk\<版本>\toolchains\
      或 DevEco Studio 安装目录下的 tools\ohpm\bin\
   2. 将上述路径添加到系统环境变量 PATH 中
   3. 重启 DevEco Studio 的 Terminal

───────────────────────────────────────────────────
Q11: 项目路径包含中文怎么办？
───────────────────────────────────────────────────
A: 将 MyApplication3 文件夹复制到纯英文路径下：
   例如 D:\HarmonyProjects\MyApplication3\
   然后用 DevEco Studio 重新打开该路径下的项目。

───────────────────────────────────────────────────
Q12: 模拟器一直卡在启动画面
───────────────────────────────────────────────────
A: 可能原因：
   1. 电脑性能不足：关闭其他程序，确保至少 4GB 空闲内存
   2. 首次启动较慢：等待 3-5 分钟
   3. 模拟器系统镜像损坏：删除模拟器重新创建
   4. 如果持续无法启动，改用远程模拟器（见第九章方式B）
